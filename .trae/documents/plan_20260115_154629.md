## 修复编辑对话框没有读取原有内容的问题

### 问题分析
当用户通过右键菜单点击"编辑"项时，OnCommand函数中的index和isDoneList都是0，导致m_dataManager.GetItemAt(0, false)返回nullptr，然后dlg(*pItem)创建了一个空的任务项，导致编辑对话框显示为空。

### 解决方案

1. **修改右键菜单处理**
   - 将右键菜单中的"编辑"项的命令ID从ID_TODO_EDIT改为ID_CONTEXT_EDIT
   - 确保传递正确的index和isDoneList信息

2. **修改OnCommand函数**
   - 添加对ID_CONTEXT_EDIT命令的处理
   - 使用m_nSelectedIndex和m_bSelectedIsDone来获取当前选中的任务项
   - 确保调用OnTodoEdit函数时传递正确的参数

3. **修改OnTodoEdit函数**
   - 添加pItem的检查，确保pItem不为nullptr
   - 如果pItem为nullptr，不创建编辑对话框，或者显示一个错误信息

### 具体实现

1. **修改ShowContextMenu函数**
   ```cpp
   // 将ID_TODO_EDIT改为ID_CONTEXT_EDIT
   if (!isDoneList) {
       menu.AppendMenu(MF_STRING, ID_CONTEXT_MARK_DONE, L"标记为完成");
       menu.AppendMenu(MF_STRING, ID_CONTEXT_EDIT, L"编辑");
   }
   ```

2. **修改OnCommand函数**
   ```cpp
   // 添加ID_CONTEXT_EDIT命令处理
   case ID_CONTEXT_EDIT:
       if (m_nSelectedIndex >= 0) {
           return OnTodoEdit(MAKELPARAM(m_nSelectedIndex, m_bSelectedIsDone ? 1 : 0), 0, 0, bHandled);
       }
       break;
   ```

3. **修改OnTodoEdit函数**
   ```cpp
   LRESULT CMainFrame::OnTodoEdit(UINT /*uMsg*/, WPARAM wParam, LPARAM /*lParam*/, BOOL& bHandled)
   {
       // 从 wParam 获取索引和是否为 Done 列表
       int index = LOWORD(wParam);
       bool isDoneList = HIWORD(wParam) != 0;

       if (index >= 0) {
           const TodoItem* pItem = m_dataManager.GetItemAt(index, isDoneList);
           if (pItem) {
               // 使用 AddTodoDlg 对话框来编辑任务，并传入原任务数据
               CAddTodoDlg dlg(*pItem);
               INT_PTR nRet = dlg.DoModal();
               // ... 其他代码
           }
       }

       bHandled = TRUE;
       return 0;
   }
   ```

这些修改将确保当用户通过右键菜单点击"编辑"项时，传递正确的index和isDoneList信息，编辑对话框就会显示原有任务内容。
